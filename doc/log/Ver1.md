# PixelLens Ver1 構想メモ

## 概要

`PixelLens` を `DPI変更ツール` から、`画像ファイルの内部構造を可視化する macOS アプリ` へ発展させる構想である。

目的は実用的な画像編集ソフトではなく、画像ファイルをデータレベルで観察し、構造を理解できるツールにすること。

想定する体験:

* 画像を開く
* ファイル内部の構造を一覧表示する
* 特定のセグメントやタグを選ぶ
* 対応するバイト列がハイライトされる
* 逆にバイト列を選ぶと、対応する構造要素がわかる

---

## 方向転換の要点

Ver0 では以下が中心だった。

* JPEG の DPI 確認
* TIFF / EXIF 系解像度の更新
* 印刷サイズ確認

Ver1 では中心を以下へ移す。

* 画像ファイルの内部構造の可視化
* バイナリと意味情報の対応表示
* JPEG / PNG / TIFF などの構造理解
* 将来的なデータレベル編集

つまり、`メタデータ編集ツール` ではなく `画像ファイル構造インスペクタ` に寄せる。

---

## 想定する主機能

### 1. ファイル構造ツリー表示

例:

* JPEG
  * SOI
  * APP0 JFIF
  * APP1 EXIF
  * DQT
  * DHT
  * SOS
  * EOI
* PNG
  * IHDR
  * pHYs
  * IDAT
  * IEND
* TIFF
  * Header
  * IFD
  * Tag entries

### 2. バイナリ表示

* 16進表示
* ASCII 表示
* オフセット表示
* 範囲選択

### 3. 相互ハイライト

* 構造ツリーで要素を選ぶと、対応するバイト範囲をハイライト
* バイト範囲を選ぶと、対応する構造要素をハイライト

### 4. デコード済み情報表示

例:

* offset
* length
* marker / chunk type
* raw bytes
* decoded value
* 説明文

### 5. 画像プレビュー

* 元画像の表示
* 構造確認中も画像自体を見られるようにする

### 6. 将来の編集機能

初期段階では read-only を優先するが、将来的には以下を検討する。

* 特定タグの書き換え
* メタデータ値の編集
* セグメント再保存
* 変更前後 diff 表示

## 技術スタックの変更点

### Ver0 の重心

* UI: `SwiftUI`
* 画像I/O: `ImageIO`
* アプリ層: `Swift`
* コア計算: `C++`

### Ver1 の重心

* UI: `SwiftUI` を継続
* 高度なビュー制御: 必要に応じて `AppKit` 追加
* ファイル読込: `Data`, `FileHandle`, 必要なら `mmap`
* 構造解析コア: `C++`
* 画像プレビュー補助: `ImageIO`
* ブリッジ層: `Swift`

重要な変化:

* `ImageIO中心` から `C++ parser中心` へ重心が移る
* `画像編集アプリ` ではなく `バイナリ構造ビューア` に近づく

---

## 新しいコア設計の方向

### コア責務

`C++` 側で以下を担う。

* JPEG セグメント解析
* PNG chunk 解析
* TIFF IFD 解析
* offset / length の計算
* バイト範囲と意味情報の対応付け

### データモデル

Ver0 の `ImageInfo` 中心では不足するため、Ver1 では木構造モデルが必要になる。

例:

* FileNode
* SegmentNode
* ChunkNode
* TagNode

各ノードが持つ情報:

* name
* kind
* offset
* length
* children
* raw bytes
* decoded value
* description

## UI 構成案

### 画面レイアウト案

* 左: 構造ツリー
* 中央: 画像プレビュー
* 右: hex / byte viewer
* 下部または右下: inspector

inspector に表示する内容:

* ノード名
* オフセット
* 長さ
* デコード値
* 補足説明

## 開発フェーズ案

### Phase 1

* JPEG read-only 構造解析
* 構造ツリー表示
* hex viewer 表示
* 構造選択時のハイライト

### Phase 2

* バイト選択から構造ノードの逆引き
* inspector の強化
* プレビューとの連携改善

### Phase 3

* PNG 対応
* TIFF 対応

### Phase 4

* 安全な部分編集
* 保存前 diff
* 編集結果検証

---

## 設計上の注意

* 最初から編集機能まで入れず、まずは read-only で成立させる
* 破損しやすいので parser と serializer は分ける
* `ImageIO` は補助利用にとどめる
* 将来の Windows 展開を考えるなら、構造解析はできるだけ `C++` に寄せる

---

## Ver1 時点の結論

この構想は技術的に可能である。

ただし、Ver0 の `DPI変更ツール` の延長というより、実質的には以下への再設計になる。

* `画像ファイル内部構造ビューア`
* `画像メタデータインスペクタ`
* `将来的に軽いバイナリエディタ的要素を持つアプリ`

したがって、今後の `PixelLens` は `ImageIO ベースの DPI ツール` から、`C++ parser ベースの構造可視化アプリ` へ設計重心を移す。
